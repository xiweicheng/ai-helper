> 适合平台：少数派、小红书、即刻、NGA  
> 关键词：效率工具、浏览器插件、AI 助手、自动化

## 开篇：一个真实的工作日

早上 9:30，你打开浏览器，今天的待办是这样的：

- 从竞品网站提取 3 个页面的定价表格，汇总成 Excel
- 把产品文档从网页版整理成 Markdown 格式
- 翻译一篇英文技术博客的关键段落
- 检查 PR 中的代码变更是否有安全隐患

如果手动完成，这些任务大概需要 **2-3 小时**。

装上 AI Helper 之后，每项任务你只需要 **说一句话**。

---

## 场景一：数据分析师的三秒取数

```
用户："把这个页面上的价格表提取出来，整理成对比表格"
```

AI Helper 自动完成：
1. 识别页面中的 `<table>` 元素
2. 提取表头和数据行
3. 格式化为结构化表格
4. 支持一键导出 CSV/Excel

不只是简单的表格——它还能识别非标准表格布局（div 模拟的表格）、处理合并单元格、自动推断数据类型。

---

## 场景二：开发者的代码审查加速器

切换到"代码审查专家"助手：

```
用户："审查当前 PR 的代码变更，重点关注安全问题和性能问题"
```

助手会：
1. 读取页面上的 diff 内容
2. 按安全 → 性能 → 最佳实践的顺序逐项审查
3. 输出结构化的审查报告，标注严重等级
4. 给出具体的修改建议和代码示例

## 场景三：产品经理的竞品调研利器

```
用户："帮我把这 3 个竞品网站的功能特点提取出来，整理成功能对比矩阵"
```

AI Helper 可以：
1. 依次打开 3 个标签页
2. 提取每个页面的功能描述
3. 自动对比分析
4. 输出 Markdown 格式的功能对比矩阵
5. 渲染 Mermaid 图表可视化

## 场景四：运营人员的内容批处理

```
用户："帮我把这篇文章翻译成英文，保持技术文档的专业风格"
```

在任意页面选中文本 → 浮动工具栏 → 翻译 → 完成。

更进一步：
- 选中一段话 → "解释这段代码的逻辑"
- 选中术语 → "这个术语的中文含义是什么"
- 选中数据 → "帮我把这些数据画成柱状图"

## 如何开始？

1. **安装**：从 GitHub 下载，`npm install && npm run build`，Chrome 加载 `dist` 目录
2. **配置**：填入你的 API Key（支持 DeepSeek、OpenAI、通义千问等）
3. **使用**：`Ctrl+Shift+Y` 打开侧边栏，开始对话

不需要付费订阅，不需要注册账号，API 费用直接付给模型厂商（很多国产模型有免费额度）。

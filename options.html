💡 配置说明

对话配置控制对话历史的存储和显示行为。
• 输入历史：用于快速重发历史输入，方便重复提问
• 对话轮数：影响上下文记忆长度，过多会增加 API 成本
• 上下文窗口：可在添加自定义模型时单独设置，或由系统根据模型名自动推断
• 执行日志：启用后可在消息底部查看 API 调用和工具执行的详细记录
• 建议根据实际需求平衡存储成本和使用体验

💡 反思配置说明

后置反思：在推理循环结束后，AI 会对最终结果进行自我评估。如果评分低于阈值，会自动修订或重新执行。
工具级反思：在每次工具调用后，AI 会检查执行结果是否符合预期，遇到错误或异常时自动调整策略。
子任务反思：对于复杂任务拆解后的子任务进行单独评估，默认关闭。
• 提高质量阈值会减少反思次数，但可能降低结果质量
• 降低温度系数会使反思更保守、更一致
• 启用反思会增加 API 调用次数，建议根据需求权衡

💡 使用说明

方式一：NPM 全局安装（推荐）
1. npm install -g ai-helper-agent（要求 Node.js ≥ 18）
2. ai-helper-agent start
3. 终端显示 4 位配对码，在此页面填入并点击"连接"

方式二：本地源码运行（开发调试）
1. cd agent && node bin/agent.js start
2. 终端显示 4 位配对码，在此页面填入并点击"连接"

后台运行：
　方式一：ai-helper-agent start --background
　方式二：node bin/agent.js start --background
更多命令：ai-helper-agent help

⚠️ 安全提示

代理服务仅监听 127.0.0.1，外部网络无法访问。
文件操作限制在配置的工作目录内，不可越权访问。
危险命令（如 rm -rf /）会被自动拦截。

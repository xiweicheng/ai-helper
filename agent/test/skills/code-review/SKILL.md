# 代码审查

## 何时使用
- 用户要求审查代码变更
- 用户提交了 pull request 或 diff
- 用户询问代码质量和安全问题
- 用户想要最佳实践建议

## 执行步骤
1. 使用 `agent_read_file` 读取用户指定的代码文件
2. 检查以下方面：
   - **代码质量**：命名规范、函数长度、复杂度
   - **安全性**：SQL注入、XSS、敏感信息泄露
   - **最佳实践**：错误处理、类型安全、模块化
   - **性能**：潜在的性能瓶颈
3. 如有脚本可用，运行 `scripts/check-style.sh` 检查代码风格
4. 汇总审查结果，按严重程度分类呈现

## 输出格式
```
## 代码审查报告

### 严重问题 (Critical)
- ...

### 警告 (Warning)
- ...

### 建议 (Suggestion)
- ...

### 总体评分
X/10
```

# 修改后自动构建

每次完成代码修改（包括但不限于：bug 修复、功能新增、样式调整、配置变更、重构优化等任何代码改动）之后，在告知用户结果之前，必须自动运行以下命令重新构建项目：

```bash
npm run build
```

## 适用场景

- 所有修改了源代码文件的任务完成后
- 所有修改了配置文件（package.json、vite.config.js 等）的任务完成后
- 所有修改了 HTML/CSS/JS/TS 资源文件的任务完成后
- bug 修复完成后
- 功能开发完成后

## 不适用场景

- 纯文档修改（README、注释等不影响构建产物的改动）
- 纯代码分析、方案讨论、不涉及文件修改的任务
- 读取文件、搜索代码等只读操作

## 要求

1. 构建命令必须在项目根目录下执行
2. 构建成功后再告知用户结果
3. 如果构建失败，必须先修复构建错误，再重新构建，直到构建成功
4. 构建结果（成功/失败、关键错误信息）应作为任务完成汇报的一部分

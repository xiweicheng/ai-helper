# 修改后自动构建

**核心规则**：每次代码修改完成后、告知用户结果前，必须运行 `npm run build:silent`（静默构建，仅失败时输出日志）。

## 触发条件（任一满足即触发）

- 修改了 `src/` 目录下的任何源文件（`.js`、`.html`、`.css`）
- 修改了配置文件（`package.json`、`vite.config.js`）
- bug 修复、功能新增、样式调整、重构优化

## 不触发条件

- 纯文档修改、只读分析、方案讨论、不涉及文件修改的任务

## 执行要求

1. 在项目根目录执行构建命令
2. 构建失败 → 先修复错误 → 重新构建 → 直到成功
3. 构建结果必须作为任务完成汇报的一部分
